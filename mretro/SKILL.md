# /mretro

You made bets when you launched. Time to find out which ones were right.

/mretro reads your `ASSUMPTIONS.md`, asks for the actual numbers you have,
and scores each assumption against signal — not gut feel. The output is a
scored assumption log and a prioritized list of what to change next.

## Preamble

```bash
_DATE=$(date +%Y-%m-%d)
_LAUNCH_DIR=$(ls -td marketing/launch/*/ 2>/dev/null | head -1)
_ASSUMPTIONS=""
[ -n "$_LAUNCH_DIR" ] && [ -f "${_LAUNCH_DIR}ASSUMPTIONS.md" ] && _ASSUMPTIONS="${_LAUNCH_DIR}ASSUMPTIONS.md"
echo "LATEST LAUNCH: ${_LAUNCH_DIR:-none}"
echo "ASSUMPTIONS: ${_ASSUMPTIONS:-none found}"
_RETRO_OUT="marketing/retro/$_DATE"
mkdir -p "$_RETRO_OUT"
echo "Retro output: $_RETRO_OUT"

# Check for existing UTM/analytics exports
UTM_FILE=""
for f in marketing/analytics/utm*.csv marketing/analytics/utm*.json analytics/utm*.csv 2>/dev/null; do
  [ -f "$f" ] && UTM_FILE="$f" && break
done
[ -n "$UTM_FILE" ] && echo "UTM DATA: $UTM_FILE" || echo "UTM DATA: none found"
```

## Phase 1: Load the Bets

Read `$_ASSUMPTIONS`. If not found, search `marketing/launch/*/ASSUMPTIONS.md`
for the most recent file. If none found, ask user to point to it.

List each assumption ID (A1, A2, B1, etc.) and its test definition. These are
the bets you are now scoring.

## Phase 2: Data Collection

Ask the user for each data source. **This is a structured intake — do not
guess, do not interpolate, do not skip to scoring without data.**

Present as a checklist. For each item, say what it tells you:

```
DATA CHECKLIST

GitHub signals (tells us: ICP accuracy, channel performance)
  [ ] GitHub star count: _____
  [ ] Stars in first 7 days: _____
  [ ] Issues filed in first 7 days: _____
  [ ] Describe the people filing issues (titles, context): _____

Channel signals (tells us: which channel assumption held)
  [ ] Where did installs come from? (paste referrer data, UTM params, or just
      tell me: "most from HN", "most from Reddit", "mostly direct")
  [ ] Did any specific post/thread drive a spike? Which?

Engagement signals (tells us: which hypothesis resonated)
  [ ] Which hypothesis copy got the most discussion/replies? A / B / C / unknown
  [ ] Did anyone quote or screenshot specific lines from the launch copy? What?
  [ ] Any unexpected audiences showing up? (wrong ICP, wrong use case)

Conversion signals (tells us: install friction, ICP accuracy)
  [ ] Views/clicks to the repo or landing page: _____
  [ ] GitHub clone count (if visible): _____
  [ ] Any "how do I install this?" questions? (Y/N — signals A2)
  [ ] Any direct feedback about pricing/cost? (Y/N — signals D1)

Qualitative signals (tells us: what the market is actually saying)
  [ ] Paste 2-3 verbatim comments, tweets, or messages you received: _____
  [ ] Any assumptions that clearly broke? (optional — name the assumption ID)
```

If the user provides partial data, accept it and score what you can. Mark
assumptions with missing data as UNCLEAR, not CONFIRMED or REFUTED.

If UTM data file was found in preamble, read it and extract source attribution
before asking the user — pre-fill what you can.

## Phase 3: Score Each Assumption

For each assumption in `ASSUMPTIONS.md`, produce a verdict:

```
[ID] [Assumption short form]
Verdict: CONFIRMED | REFUTED | UNCLEAR
Evidence: [one sentence — the specific data point that drove this verdict]
Implication: [one sentence — what to do differently because of this]
```

**Scoring rules:**
- CONFIRMED requires at least one concrete signal matching the test defined in
  the assumption (not just "things went OK").
- REFUTED requires at least one concrete signal contradicting the test.
- UNCLEAR means the signal was absent, ambiguous, or the test period was too short.
- Do not use gut feel. If you don't have data, it's UNCLEAR.

**Special cases:**
- If an ICP assumption (A*) shows unexpected audiences — name them. They may be
  a better ICP than the original bet.
- If a channel assumption (C*) shows one channel dominating, flag it for doubling.
- If a hypothesis assumption (C3 / "which hook worked") shows a clear winner,
  say so explicitly: "Hypothesis A outperformed B and C — double down."

## Phase 4: Write the Retro

Write `marketing/retro/{date}/RETRO.md`:

```markdown
# Marketing Retro — [Product Name] — [Date]
Generated by /mretro from launch: [launch dir]
Data period: [X days since launch]

## Scorecard

| ID | Assumption | Verdict | Evidence |
|----|-----------|---------|----------|
| A1 | [short form] | CONFIRMED | [one sentence] |
| B1 | [short form] | REFUTED | [one sentence] |
| C1 | [short form] | UNCLEAR | [why — missing data] |
...

## What held

[CONFIRMED assumptions — what this tells us to double down on]

## What broke

[REFUTED assumptions — what this tells us to stop or change]

## Surprises

[Anything that happened that no assumption predicted — unexpected audiences,
unexpected channels, unexpected language that resonated]

## Revised ICP (if any)

[If the data suggests the actual buyer is different from A1 — say so here.
Quote the evidence. This is the most important output of the retro.]

## Next actions

Prioritized list, each tied to a specific assumption verdict:
1. [Action] — because [A1/B2/etc.] was [CONFIRMED/REFUTED]
2. ...

## Assumptions to carry forward

[Which UNCLEAR assumptions still need testing? What would generate signal?]

## Data gaps for next launch

[What would you instrument differently if you ran this again?]
```

## Phase 5: Update ASSUMPTIONS.md

For each scored assumption, append the verdict to the source file:

```
- [x] **A1:** [original text]
      Evidence: [original evidence]
      Test: [original test]
      **RETRO ({date}):** CONFIRMED — [one sentence of evidence]
```

Or for REFUTED:
```
- [x] **B1:** [original text]
      **RETRO ({date}):** REFUTED — [evidence]. Implication: [what to change].
```

Preserve the full original text. Only append — never overwrite.

## Completion

```bash
echo ""
echo "✦ /mretro complete: $_RETRO_OUT"
echo ""
echo "  RETRO.md             — scored assumption log + next actions"
echo ""
echo "Assumptions updated in: $_ASSUMPTIONS"
echo ""
_CONFIRMED=$(grep -c "RETRO.*CONFIRMED" "$_ASSUMPTIONS" 2>/dev/null || echo 0)
_REFUTED=$(grep -c "RETRO.*REFUTED" "$_ASSUMPTIONS" 2>/dev/null || echo 0)
_UNCLEAR=$(grep -c "RETRO.*UNCLEAR" "$_ASSUMPTIONS" 2>/dev/null || echo 0)
echo "Score: $_CONFIRMED CONFIRMED / $_REFUTED REFUTED / $_UNCLEAR UNCLEAR"
```

Report: DONE if all assumptions scored and RETRO.md written.
Report: DONE_WITH_CONCERNS if data was incomplete (list which assumptions are UNCLEAR and why).
Report: NEEDS_CONTEXT if no ASSUMPTIONS.md was found and user couldn't provide one.
